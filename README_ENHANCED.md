## 🏗️ Repository Structure

This is a **monorepo with independent publishing**:

```
SDK/                                    # Main repository (source of truth)
├── projects/                          # All SDKs in one workspace
│   ├── api-contract-drift-detector/   # SDK 1
│   ├── ai-powered-migration-generator/ # SDK 2
│   ├── android-ai-provider-core/      # Android SDK 1
│   └── ... (23 total)
├── .github/workflows/                 # CI/CD pipelines
│   ├── publish-web-sdks.yml          # npm auto-publish (all 15 web SDKs)
│   └── publish-android-sdks.yml      # Maven Central auto-publish (all 8 Android SDKs)
├── build.gradle.kts                   # Root Gradle build for Android SDKs
├── settings.gradle.kts                # Gradle multi-project config
├── gradle.properties                  # Gradle + Android config
└── package.json                       # npm workspace root config
```

## 🚀 Publishing Architecture

### Web SDKs (npm)

- **Trigger**: Push to `main` + `projects/*/` path changes OR manual workflow dispatch
- **Build**: npm install → test → build per SDK
- **Publish**: npm publish `@amitgaikwad37/<name>` to npmjs.org
- **Versioning**: Automatic via git tags (v*) or run-number for snapshots
- **Independence**: Failure in one SDK doesn't block others (fail-fast: false)

### Android SDKs (Maven Central)

- **Trigger**: Push to `main` + `projects/android-*/` path changes OR manual workflow dispatch
- **Build**: Gradle build → test per SDK module
- **Publish**: Gradle publish to Maven Central Staging
- **Signing**: PGP signing with SIGNING_KEY, SIGNING_PASSWORD env vars
- **Independence**: Failure in one SDK doesn't block others (fail-fast: false)

## 🔄 Development Workflow

1. **Local Development**: Work in `projects/<sdk>/`
2. **Testing**: `npm test` (web) or `gradle :module:test` (android)
3. **Publishing**: Push to main → workflows auto-publish → appears in registry
4. **Independent SDKs**: Each SDK builds/publishes independently; failures don't cascade

## 🔐 Credentials & Secrets

### Web SDK Publishing (npm)
- `NPM_TOKEN`: npm automation token with 2FA bypass
- Stored in GitHub org secrets

### Android SDK Publishing (Maven Central)
- `CENTRAL_PORTAL_USERNAME` / `CENTRAL_PORTAL_PASSWORD`: Portal API credentials
- OR `SONATYPE_USERNAME` / `SONATYPE_PASSWORD`: Legacy Sonatype credentials
- `SIGNING_KEY`, `SIGNING_KEY_ID`, `SIGNING_PASSWORD`: PGP signing
- Stored in GitHub org secrets
